RouteForce Goes Worldwide Routing and AppExchange Ready

Two concrete milestones this week: a routing engine that covers the entire planet, and an application ready for the Salesforce marketplace.

A Worldwide Routing Engine

Until now, RouteForce computed routes within France and neighboring countries. That works for most cases, but it falls short when a client operates in Senegal, the UAE, or South America.

This week, we built a planetary road graph based on OpenRouteService and OpenStreetMap data. The numbers:

DataValue
Source fileplanet-latest.osm.pbf (78 GB)
Road nodes309 million

Build time~3 days on high-performance infrastructure
InfrastructureDedicated VPS, 64 GB RAM, SSD

Starting next week, a field rep using RouteForce can compute an optimized route anywhere in the world: Dakar, Dubai, São Paulo, or Melbourne. Same engine, same accuracy, same speed.

Why It Matters

Most Salesforce routing solutions rely on Google Maps or third-party services billed per API call. That means variable costs, request limits, and dependency on an external provider.

RouteForce runs a self-hosted open source engine. Routing cost is fixed and included in the subscription, regardless of calculation volume. No surprise bills, no quotas to monitor.

AppExchange: Where We Stand

RouteForce is a Salesforce managed package (2GP). We just cleared the security steps required to submit the application on the AppExchange:

  • Salesforce Code Analyzer passed — only 2 findings, both documented as false positives (customer-editable CMDT fields, by design).
  • API keys migrated to Protected Custom Settings. No secrets hardcoded in the source.
  • Top 20 AppExchange vulnerability audit — 18/20 clean, 2 documented false positives.
  • 258 Apex tests, 77% code coverage. Zero failing tests.
  • TLS verified: all communications encrypted with AES-256-GCM.
  • Permissions: two PermissionSets (User and Admin), FLS enforced on all objects.

Submission is planned in the coming weeks. Salesforce's review process typically takes 6 to 9 weeks. Target: published on the AppExchange by May 2026.
