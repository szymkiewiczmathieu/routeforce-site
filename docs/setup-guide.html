Setup Guide

Install RouteForce, assign access, configure core settings, and validate your first rollout.

1. Prerequisites

  • Salesforce org (Enterprise, Unlimited, or Developer Edition)
  • System Administrator profile
  • Translation Workbench enabled — Setup > Translation Workbench > Enable
  • Languages activated: English, French, Spanish (if multilingual)
  • Optional: Account Teams enabled — required for the "My Account Team" filter
  • Optional: Team Selling enabled — required for the "My Opportunity Team" filter

2. Install the Package

Install the RouteForce managed package using the link matching your environment:

EnvironmentLink
ProductionInstall v4.3.0 — Production
SandboxInstall v4.3.0 — Sandbox

Select Install for Admins Only (recommended), then assign permission sets manually.

3. Assign Permission Sets

RouteForce ships two permission sets:

Permission SetTarget UsersCapabilities
RouteForce AdminSalesforce AdminsFull access + Configuration tab + Licence tab + modifyAllRecords
RouteForce UserSales RepsStandard access to map, routes, history, filters, visit reports

Assign via Setup > Permission Sets or via CLI:

sf org assign permset --name RouteForce_Admin --target-org <your-org>
sf org assign permset --name RouteForce_User --target-org <your-org>

Enable "Run Flows" Permission

RouteForce uses Screen Flows (e.g., Create Event, Custom Actions). The Standard User profile does not have Run Flows enabled by default. Without it, users see:

Error: "You do not have the level of access necessary to perform the operation you requested."

This permission must be configured manually by the admin:

Option A — Enable on the Profile: Setup > Profiles > Standard User > System Permissions > enable Run Flows

Option B — Create a Permission Set: Setup > Permission Sets > New > System Permissions > enable Run Flows > assign to all RouteForce users

4. Run the Onboarding Wizard

  1. Open the RouteForce app from the App Launcher
  2. Navigate to the Configuration tab
  3. Launch the RF_Onboarding screen flow
  4. Enter your API keys:
FieldRequiredDescription
Licence KeyYesYour RouteForce licence key (provided by RouteForce)
ORS API KeyYesAPI key for the VPS routing server
Fallback API KeyNoHeiGIT fallback key (used when VPS is unavailable)
Note: Keys are stored securely in a protected Custom Setting (InternalConfig__c), invisible to standard users.

After saving keys, the wizard directs you to configure application settings via Custom Metadata Types:

Setup → Custom Metadata Types → RouteForce Config → Manage Records → Default

Out-of-the-Box Defaults

  • Leads & Opportunities enabled (EnableLeads__c, EnableOpportunities__c = true)
  • Filter fields pre-configured: Account.Industry, Account.Type
  • History naming enabled — users are prompted to name saved routes
  • Ownership defaults to "My Accounts/Leads/Opportunities" per user

Visit Report Settings (Recommended)

FieldDescription
VisitAlertDays__cDays after which a record is flagged as overdue. Markers show a red alert badge. Set to 0 to disable.
VisitResultOptions__cComma-separated picklist values for the visit result field (e.g., Successful,Follow-up needed,No contact). Leave blank for free text.

7. Custom Action Buttons (Optional)

Add custom buttons to marker popups that launch your own Screen Flows.

Step 1 — Create a Screen Flow

  1. Create a Screen Flow with a recordId Text input variable (mark as "Available for Input")
  2. Add a Screen element containing your LWC component
  3. Pass {!recordId} to the component's recordId attribute
  4. Activate the flow

Step 2 — Register the Action

Setup > Custom Metadata Types > RouteForce Action > Manage Records > New

FieldDescription
LabelButton text (e.g., "Data Entry")
Flow API NameYour flow API name
Object TypeWhich markers show this button: All, Account, Lead, Opportunity
Display ContextAll, Popup, or EventPanel
Button ColorHex color (e.g., #F59E0B)
Iconaction, edit, check, chart, list, or star
Filter ConditionOptional. Syntax: FieldName=Value, FieldName!=Value, FieldName=Val1,Val2

8. Event Layout — Visit Report Field

RouteForce adds a Visit Report (VisitReport__c) lookup field on the Activity object, linking each Event to the Visit Report created during check-in.

To make this field visible on the Event record page: Setup > Object Manager > Event > Page Layouts > drag "Visit Report" onto the layout

Note: This step is optional but recommended. Without it, users won't see the link to the Visit Report from their Event records.

9. Test the Installation

  1. Open the RouteForce tab
  2. Verify the map loads with your geocoded Accounts
  3. Try filtering by Industry or Type
  4. Enable Route mode and add stops
  5. Optimize a route
  6. Check the Licence tab — should show "Active"

10. Logging & Log Purge

RouteForce logs application events in the RouteForceLog__c custom object. No setup required — logs are created automatically.

LevelExamples
INFOBatch completion, configuration changes, route saves
WARNAPI fallback triggered, unexpected data format
ERRORCallout failures, DML errors, licence validation failures

Schedule the automatic log purge (runs daily at 2:00 AM, deletes logs older than 30 days):

System.schedule('RouteForce Log Purge', '0 0 2 * * ?', new routeforce.LogPurgeSchedulable());

Custom retention period (e.g., 90 days):

System.schedule('RouteForce Log Purge', '0 0 2 * * ?', new routeforce.LogPurgeSchedulable(90));

11. Multilingual Support

RouteForce supports English, French, and Spanish out of the box. The UI language follows the user's Salesforce language preference.

To verify: change your language in Settings > Language & Time Zone and refresh the app.
