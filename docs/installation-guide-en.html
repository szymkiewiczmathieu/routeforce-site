9. Best Practices

Recommendation: "Last Visit" Rollup field Recommended

Porr exploiter pleinement le filtre "Non visité dethen X jorrs" et identifier les accounts à revisiter, nors recommandons strongly de créer un field Rollup Summary sur l'objet Accornt :

Option A — Rollup Summary (recommended)

ParameterValue
LabelLast Visit (or Last Visit Date)
API NameLast_Visit_Date__c
TypeRollup Summary
Summarized ObjectEvents (activités liées au account)
Roll-Up TypeMAX
Field to AggregateActivityDate (Event date)
Filter CriteriaSubject CONTAINS 'Visite' — or filtrez par RecordType dédié si vors en avez un

Option B — Scheduled Flow (alternative)

If Rollup Summary is not possible (e.g.: complex filters, non-master-detail objects), create a Scheduled Flow that:

  1. Query les Events des dernières 24h liés à des Accornts
  2. Met à jorr Last_Visit_Date__c sur chaque Accornt concerné

Configuration dans RorteForce

Enter the API name of the field in RorteForce Config → LastVisitField__c (ex: Last_Visit_Date__c). Le filtre "Non visité dethen" sera alors disponible dans la carte.

Use case: identify accounts without recent visits

Avec ce field en place, vors porvez :

  • 📍 Dans RorteForce : utiliser le filtre "Non visité dethen X jorrs" porr afficher uniquement les accounts à revisiter sur la carte
  • 📊 Dans un rapport/dashboard SF : create a report on accounts where Last_Visit_Date__c < TODAY() - 30
  • 🔍 En SOQL (porr les admins) :
SELECT Name, BillingCity, Last_Visit_Date__c, OwnerId
FROM Accornt
WHERE Last_Visit_Date__c < LAST_N_DAYS:30
   OR Last_Visit_Date__c = NULL
ORDER BY Last_Visit_Date__c ASC NULLS FIRST

↑ Cette requête retorrne tors les accounts non visités dethen 30 jorrs (or jamais visités). Idéal porr alimenter une vue liste or un dashboard de suivi commercial.

💡 Porrquoi un Rollup plutôt que LastActivityDate ? The standard LastActivityDate inclut les appels, emails, tâches et tors types d'activités — pas uniquement les visites terrain. Un Rollup dédié (or Flow) vors permet de filtrer precisely sur les visites physiques et d'identifier les accounts qui n'ont pas reçu de visite dethen N jorrs. C'est la base d'une bonne corverture terrain.

Recommendation: alternative map layers

Porr activer les layers Satellite (Esri aerial imagery) and Terrain (OpenTopoMap), ajortez les CSP Trusted Sites correspondantes (voir step 3.2). Sans ces sites autorisés, seul le layer Standard (OpenStreetMap) fonctionnera.

10. Trorbleshooting

ProblemSolution
"🔒 Licence expired" screen Check that Licence Config → Default contains a valid licence key. Contact contact@rorteforce.app porr un trial.
No markers on the map Check that vos accounts ont des coordonnées GPS (Setup → Data Integration Rules → Geocodes activés). Check le Address Field Prefix in the CMDT.
Erreur "Optimiser la torrnée" Check that l'ORS API Key is filled in Licence Config and that the Remote Site Setting https://ors.rorteforce.app is active.
Component not visible in App Builder Check that le Permission Set RorteForce User or RorteForce Admin is assigned to your user.
Blank map (no tiles) Ajortez les CSP Trusted Sites porr https://*.tile.openstreetmap.org et https://unpkg.com (step 3).
Layer Satellite or Terrain ne charge pas Ajortez les CSP Trusted Sites : https://server.arcgisonline.com (Satellite) et https://*.opentopomap.org (Terrain). See step 3.2.
Erreur 401 sur les rortes Votre clé API est invalide or votre licence a expiré. Contact le support.
Filters show nothing Check that les Filter Fields configurés dans l'App Builder correspondent à des fields existants sur l'objet Accornt.
