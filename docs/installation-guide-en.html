5. Licence Configuration

RouteForce uses a licence system to activate the component. Keys are provided by support after registration (14-day trial or subscription).

Step 5.1 — Enter the licence

Navigate to the Custom Metadata Type LicenceConfig:

Setup → Custom Metadata Types → "Licence Config" → Manage Records → Edit "Default"
FieldValueDescription
Licence Key (provided by support) Obligatoire RouteForce licence key. Without this key, the component displays a licence expired screen.
ORS API Key (provided by support) Obligatoire Authentication key for the optimization server. Without this key, route calculations and optimization will not work.
💡 Free Trial : Contact contact@routeforce.app for a free 14-day trial. You will receive your licence key and API key via secure email (one-time link, auto-destroyed after reading).
💡 Licence Activation : Licence validation is automatic. Once the key is entered, RouteForce verifies validity with our servers and activates the application. If the key changes, the update takes effect within 24h.

6. Custom Metadata Configuration

All RouteForce configuration is centralized in a single Custom Metadata Type : RouteForceConfig. A Default record is provided with default values. All fields are SubscriberControlled: you can modify them freely.

Setup → Custom Metadata Types → "RouteForce Config" → Manage Records → Edit "Default"

6.1 — Configuration Fields

6.2 — Per-page configuration (App Builder override)

Configurable properties in the App Builder (step 7) take priority over Custom Metadata. This allows different configurations per page (ex : Billing on the Account page, Shipping on a dedicated page).

7. App Builder Configuration

Step 7.1 — Add the component to a Lightning page

  1. Open a Lightning page in the App Builder (Account page, Home or App Page)
  2. Search for "routeforce" in the components panel (left side)
  3. Drag and drop the component onto the page
  4. Configure the properties in the right panel (see below)
  5. Click Save then Activate

7.2 — Configurable Properties

PropertyDescriptionExample

💡 Dynamic Filters : Filters automatically adapt to the field type (picklist → dropdown, boolean → toggle, multipicklist → multi-select). You can filter on any standard or custom field on the Account object.

Step 7.3 — Add the RouteForce tab to the navigation bar

Recommandé The RouteForce tab provides access to the full map (all accounts view). Add it to your Lightning application:

  1. Go to Setup → App Manager
  2. Click Edit on your application (e.g.: Sales)
  3. Go to the Navigation Items tab
  4. Add RouteForce to the selected items
  5. Position it where desired and click Save

Step 7.4 — Add the Map tab on the Account page

Recommandé To see the map centered on a specific account, add a tab to the Account page:

  1. Open the Account page in the App Builder
  2. Add a Tabs component if absent, or use existing tabs
  3. Create a new tab named "Map"
  4. Drag and drop the RouteForce component into this tab
  5. Click Save then Activate

💡 The component automatically centers on the current account when placed on an Account page. Nearby accounts are visible on the map, making it easy to plan grouped visits.

⚡ Recommandation performances : Privilégiez le placement du composant dans un onglet "Map" sur la page Account plutôt que sur une page standalone (tab RouteForce). Sur une page Account, seuls les comptes à proximité sont chargés. Sur la page standalone, tous les comptes géolocalisés de votre org sont récupérés (potentiellement des milliers), ce qui peut ralentir le chargement initial. L'onglet Account offre des performances optimales.

9. Best Practices

Recommendation: "Last Visit" Rollup field Recommandé

To fully leverage the filter "Not visited in X days" and identify accounts to revisit, we strongly recommend creating a Rollup Summary field on the Account object:

Option A — Rollup Summary (recommended)

ParameterValue
LabelLast Visit
API NameLast_Visit_Date__c
TypeRollup Summary
Summarized ObjectEvents (activities related to the account)
Roll-Up TypeMAX
Field to AggregateActivityDate (Event date)
Filter CriteriaSubject CONTAINS 'Visite' — or filter by dedicated RecordType if you have one

Option B — Scheduled Flow (alternative)

If a Rollup Summary is not possible (e.g.: complex filters, non-master-detail objects), create a daily Scheduled Flow that:

  1. Queries Events from the last 24h related to Accounts
  2. Updates Last_Visit_Date__c on each relevant Account

RouteForce Configuration

Enter the API field name in RouteForce Config → LastVisitField__c (ex: Last_Visit_Date__c). The "Not visited since" filter will then be available in the map.

Use case: identify accounts without recent visits

With this field in place, you can:

  • 📍 Dans RouteForce : utiliser le filtre "Not visited in X days" pour afficher uniquement les comptes à revisiter sur la carte
  • 📊 In a SF report/dashboard: create a report on accounts where Last_Visit_Date__c < TODAY() - 30
  • 🔍 In SOQL (for admins):
SELECT Name, BillingCity, Last_Visit_Date__c, OwnerId
FROM Account
WHERE Last_Visit_Date__c < LAST_N_DAYS:30
   OR Last_Visit_Date__c = NULL
ORDER BY Last_Visit_Date__c ASC NULLS FIRST

↑ This query returns all accounts not visited in the last 30 days (or never visited). Ideal for populating a list view or sales tracking dashboard.

💡 Why a Rollup instead of LastActivityDate? The standard LastActivityDate field includes calls, emails, tasks and all activity types — not just field visits. A dedicated Rollup (or Flow) lets you filter precisely on physical visits and identify accounts that haven't received a visit in N days. This is the foundation of good territory coverage.

Recommendation: alternative basemaps

To enable the Satellite (Esri aerial imagery) and Terrain (OpenTopoMap) basemaps, add the corresponding CSP Trusted Sites (see step 3.2). Without these authorized sites, only the Standard (OpenStreetMap) basemap will work.

10. Troubleshooting

ProblemSolution
"🔒 Licence expired" screen Verify that Licence Config → Default contains a valid licence key. Contact contact@routeforce.app for a trial.
No markers on the map Verify your accounts have GPS coordinates (Setup → Data Integration Rules → Geocodes enabled). Check the Address Field Prefix in the CMDT.
"Optimize route" error Vérifiez que l'ORS API Key is filled in Licence Config and that the Remote Site Setting https://ors.routeforce.app is active.
Component not visible in App Builder Verify the RouteForce User or RouteForce Admin Permission Set is assigned to your user.
Blank map (no tiles) Add CSP Trusted Sites for https://*.tile.openstreetmap.org et https://unpkg.com (step 3).
Satellite or Terrain basemap not loading Add CSP Trusted Sites: https://server.arcgisonline.com (Satellite) and https://*.opentopomap.org (Terrain). See step 3.2.
401 error on routes Votre clé API est invalide ou votre licence a expiré. Contact le support.
Filters show nothing Verify the Filter Fields configured in App Builder correspond to existing fields on the Account object.
