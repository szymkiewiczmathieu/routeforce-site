User Guide

A complete guide to plan field visits, optimize routes, and track customer interactions — all from within Salesforce.

A complete guide for sales reps and managers using RouteForce to plan field visits, optimize routes, and track customer interactions — all from within Salesforce.

1. Getting Started

Opening the App

  1. Click the App Launcher (9-dot grid icon, top-left)
  2. Search for RouteForce
  3. Click to open — the map loads with your geocoded Account markers

Interface Overview

Area Location Purpose
Filter Panel Left sidebar Filter records, toggle display layers, save filter presets
Map Center Interactive map with Account, Lead, Opportunity, and Event markers
Route Panel Right sidebar Plan routes, add stops, optimize, export
Events Panel Top bar Load calendar events, check in at visits
History Panel Right sidebar View, reload, and manage past routes

Navigation Tabs

  • RouteForce — Main map interface
  • Route History — List view of all saved routes
  • Saved Filters — List view of saved filter configurations
  • Visit Reports — List view of all visit reports created during check-ins
  • Usage Analytics — Admin-only dashboard showing app usage and automation job status
  • Configuration — Admin-only settings (onboarding wizard, licence)

2. The Map

Map Controls

Control Location Action
Zoom +/− Top-left Zoom in or out
Fullscreen Top-left Toggle fullscreen map mode
My Location Top-left Center map on your current GPS position
Basemap Selector Filter panel > Display Switch between Standard, Satellite, or Terrain tiles
Scale Bottom-right Shows metric distance scale

Markers

Records with geocoded addresses appear as colored pin markers on the map:

Marker Color Record Type
Blue tones Accounts (color varies by Type field value)
Warm tones (orange/red) Opportunities (color varies by Stage)
Cool tones (teal/green) Leads (color varies by Status)
Calendar icon Events (blue = upcoming, gray = checked in)

Visit Alert Badge

When configured by your admin, markers can display a red alert badge (small red circle with a number) indicating how many records at that location are overdue for a visit. This helps you quickly identify which accounts need attention.

  • The badge appears when a record hasn't been visited within the configured threshold (e.g., 30 days)
  • Overdue records also display with highlighted styling in marker popups
  • The threshold is configured by your admin via VisitAlertDays__c — set to 0 to disable

Marker Clustering

When many markers overlap, they automatically cluster into a single circle showing the count. Zoom in to reveal individual markers.

Legend

The bottom of the map displays a color legend showing which color corresponds to which value (e.g., "Customer - Direct" = dark blue, "Prospect" = light blue). The legend updates based on the active display mode.

3. Marker Popups

Click any marker to open its popup with record details and quick actions.

Popup Contents

  • Name — Clickable link that opens the Salesforce record
  • Address — Street, city, postal code, country
  • Owner — Clickable link to the record owner
  • Last Visit — Date of the most recent visit report (if configured by admin for the record type: Account, Lead, or Opportunity)
  • Phone — Clickable phone number (tel: link) — tap to call directly on mobile, or click to launch your desktop phone app. Both Phone and MobilePhone are displayed when available
  • Custom Fields — Additional fields configured by your admin (e.g., Industry, Amount, Website)

Popup Actions

Button Action
Create Event Opens event creation for this record
Directions Opens Google Maps with directions to this location
Add to Route Adds this record as a stop in your current route (when route panel is open)
Nearby Shows nearby records within the configured radius
Custom Actions Additional buttons configured by your admin (e.g., data entry forms, workflows). These open in a fullscreen modal. See Custom Actions.

Group Popups

When multiple records share the same address (e.g., several Opportunities on one Account), clicking the marker shows a scrollable list of all records at that location, each with its own action buttons.

4. Filtering Records

Opening the Filter Panel

Click the filter icon on the left side of the map, or tap the filter area on mobile.

Display Toggles

Toggle which record types appear on the map:

  • Accounts — Show/hide Account markers
  • Leads — Show/hide Lead markers (if enabled by admin)
  • Opportunities — Show/hide Opportunity markers (if enabled by admin)
  • Events — Show/hide Event markers on the calendar layer

Account Filters

Filter Description
All Accounts Show all Accounts (no ownership filter)
My Accounts Show only Accounts you own
My Account Team Show Accounts where you are a member of the Account Team (admin must enable)
My Role Team Show Accounts owned by users in your role or any subordinate role (admin must enable)
Radius Distance slider — only show records within X km of center
Record Type Filter by Salesforce Record Type
Last Visit Filter by visit recency: "Never visited", "Not visited in 3/6/12/24 months". Applies to all 3 object types (Accounts, Leads, Opportunities). Date is automatically sourced from the most recent Visit Report
Picklist Filters Industry, Type, Rating, Ownership, etc. — depends on admin configuration

Lead Filters

Each Lead filter section has its own independent ownership filter:

Filter Description
All Leads Show all Leads (no ownership filter)
My Leads Show only Leads you own
My Role Team Show Leads owned by users in your role or any subordinate role (admin must enable)
Record Type Filter by Lead Record Type
Last Visit Filter by visit recency (automatic — based on most recent Visit Report)
Picklist Filters Lead Source, Status, Rating, etc. — depends on admin configuration

Opportunity Filters

Each Opportunity filter section has its own independent ownership filter:

Filter Description
All Opportunities Show all Opportunities (no ownership filter)
My Opportunities Show only Opportunities you own
My Opportunity Team Show Opportunities where you are a member of the Opportunity Team (admin must enable)
My Role Team Show Opportunities owned by users in your role or any subordinate role (admin must enable)
Record Type Filter by Opportunity Record Type
Last Visit Filter by visit recency (automatic — based on most recent Visit Report)
Picklist Filters Stage, Forecast Category, etc. — depends on admin configuration
Cross-object filters Filter Opportunities by their Account's fields (e.g., Industry)

Note: Each object's ownership filter is independent. For example, you can view "My Accounts" while simultaneously viewing "All Leads" and "My Opportunities".

Multi-Select Filters

Some filters display as checkbox lists instead of dropdowns, allowing you to select multiple values simultaneously (e.g., select both "Customer - Direct" AND "Prospect" for the Type filter).

Applying Filters

Click the Apply button after changing filters. The map refreshes with the filtered results.

5. Saved Filters

Save your frequently used filter combinations to reload them instantly.

Saving a Filter

  1. Configure your desired filters, display toggles, and basemap
  2. In the Saved Filters section (top of filter panel), click Save
  3. Enter a name (e.g., "My Western Region Prospects")
  4. Click Save — a confirmation toast appears

Loading a Saved Filter

  1. Open the Saved Filters dropdown
  2. Select a saved filter — all settings are instantly applied, including:
  3. Account, Lead, and Opportunity filters
  4. Display toggles (show/hide layers)
  5. Basemap selection
  6. Record type filters
  7. The map refreshes with the saved configuration

Deleting a Saved Filter

  1. Select the filter in the dropdown
  2. Click the Delete button
  3. Confirm deletion

6. Route Planning

Opening the Route Panel

Click the route icon button on the map toolbar to open the route planning panel on the right side.

Setting a Starting Point

Choose how your route begins:

Option Description
My Position Uses your current GPS coordinates
Account Search and select an Account record as the start
Lead Search and select a Lead record (if enabled)
Opportunity Search and select an Opportunity record (if enabled)

Click the corresponding pill button and, for record-based starts, use the search field to find the record.

Adding Stops

With the route panel open, click markers on the map to add them as route stops. Each click adds the record to your step list.

Steps appear in the route panel showing:

  • Step number (sequential badge)
  • Record name and city
  • Record type (Account, Lead, or Opportunity)
  • Phone — Clickable tel: link (when a contact is selected and has a phone number)

Reordering Steps

  • Drag and drop — Grab the hamburger icon (☰) on any step and drag it to a new position
  • Arrow buttons — Use the up/down arrows to move a step one position

Removing Steps

Click the trash icon on any step to remove it from the route.

Route Settings

Expand the Settings section to configure:

Setting Default Description
Route Date Today Date for the planned route
Departure Time 08:00 When you leave the starting point
Lunch Break 12:00–13:00 Break period (no visits scheduled during this time)
End of Day 18:00 Latest time for the last visit
Visit Duration 30 min Time spent at each stop
Avoid Tolls Off Route avoids toll roads
Avoid Highways Off Route avoids highways

7. Route Optimization

Optimizing Your Route

  1. Add at least 2 stops and set a starting point
  2. Click the Optimize button (blue, routing icon)
  3. Wait for the spinner — the system calculates the optimal order
  4. Steps are reordered for the shortest total travel time

After Optimization

  • Step numbers update to the optimized order (green badges)
  • Arrival times appear on each step (rounded to the nearest 15 minutes)
  • Effective arrival shows the raw calculated time (if different from rounded)
  • Route line draws on the map showing the optimized path
  • Summary displays total distance (km) and duration

Lunch Break Handling

The optimizer automatically accounts for your lunch break. If a visit would fall during lunch, the arrival is shifted to after the break.

Overflow Warning

If a step's arrival time exceeds your End of Day setting, it displays a red warning indicator. Consider removing steps or adjusting your schedule.

Manual Adjustments

After optimization, you can:

  • Click an arrival time to edit it manually (a pencil icon appears to indicate manual override)
  • Reorder steps — drag-and-drop still works post-optimization
  • Add/remove steps — then re-optimize if desired

Timezone

All arrival and departure times displayed in your route reflect your Salesforce user timezone (Settings > Language & Time Zone).

Step Limit

The routing engine supports a maximum of 50 stops per route. A warning appears if you exceed this limit, and the Optimize button is disabled.

8. Nearby Accounts

When a route exists, the Nearby section automatically shows records within a configurable radius of your route path.

Each nearby item displays:

  • Record name and type
  • Distance from the nearest route point (e.g., "2.5 km")
  • Add button (+) to include it as a new stop

When the Visit Alert feature is enabled by your admin, nearby records are sorted by visit urgency — records overdue for a visit appear first in the list, helping you prioritize the most important opportunistic stops.

This helps you discover opportunistic visits along your planned route.

9. Exporting & Saving Routes

Google Maps Export

Click Google Maps to open your optimized route in Google Maps with all waypoints. This is useful for turn-by-turn navigation on your phone.

CSV Export

Click Export CSV to download a spreadsheet with:

Column Content
Step Stop number
Account Record name
City City from address
Arrival Estimated arrival time
Contact Selected contact (if any)
Phone Contact phone number (if contact is selected)
Distance Distance from previous step
Duration Travel time from previous step

Creating Calendar Events

Click Create Visits to automatically create Salesforce Events for each stop:

  • Subject: Record name
  • Start/End Time: Based on optimized arrival + visit duration
  • Related To: Linked to the Account/Lead/Opportunity

Schedule Conflict Detection

Before creating the events, RouteForce automatically checks your calendar for time conflicts. If any planned visit overlaps with an existing event, a warning dialog appears showing:

  • The number of conflicting visits
  • Each conflicting event's subject and time range (e.g., "Team Sync (10:00 - 11:30)")

You can choose to create the visits anyway or cancel and adjust your route. This check is non-blocking: if it fails for any reason, events are created normally.

A confirmation message shows how many events were created. The daily agenda panel automatically opens with the newly created events, so you can immediately see your day's schedule.

Saving Route History

After optimization, save your route to history for future reference. If History Naming is enabled, you can give the route a custom name (e.g., "Paris North — March 11").

10. Events & Check-In

Loading Events

  1. Open the Events panel (calendar icon, top area)
  2. Select a date or use Today / This Week presets
  3. Click Load — events appear as calendar-pin markers on the map

Event Markers

  • Blue pulsing pin — Upcoming event (not yet checked in)
  • Gray pin with green checkmark — Checked-in event

Next Event Block

The next upcoming event is highlighted at the top of the events panel. Click it to automatically center and zoom the map to its location — useful for quickly navigating to your next appointment.

Event Popup

Click an event marker to see:

  • Subject (event title)
  • Time range (e.g., 09:00 – 09:30)
  • Related record (Account/Lead/Opportunity name, clickable)
  • City
  • Check-In button or Checked In status
  • Custom action buttons (if configured, e.g., "DN Entry")

Hiding Markers

When the daily agenda is open, you can click Hide Accounts in the toolbar to temporarily hide Account/Lead/Opportunity markers, making event pins easier to see. Closing the agenda panel restores them automatically.

Checking In

  1. Click Check In on an event popup
  2. The Check-In Modal opens with:
  3. Previous Visit context (if a prior visit report exists for this Account, Lead, or Opportunity) — shows the date, result, next action planned, and author of the last visit
  4. Result field — visit outcome. Depending on admin configuration, this is either a dropdown picklist (with predefined options like "Successful", "Follow-up needed") or a free text input (max 255 characters)
  5. Next Action field — what to do next after this visit
  6. Notes field (rich text if Quick Text is enabled, plain textarea otherwise)
  7. Quick Text snippets (pre-defined text blocks you can insert into notes)
  8. Fill in the result, next action, and notes
  9. Click Confirm

This creates a Visit Report record linked to the related record (Account, Lead, or Opportunity), and marks the event as checked in (the marker turns gray with a green checkmark).

GPS Location Capture

During check-in, RouteForce automatically captures your GPS position (via browser geolocation):

  • Latitude, Longitude, and Accuracy (in meters) are saved on the Visit Report
  • The GPS accuracy is displayed in the check-in modal so you can verify the reading
  • A GPS badge appears on checked-in steps in the Route History map view

Check-in Timestamp

A timestamp is appended to the Event description in the format: {prefix}: DD/MM/YYYY HH:MM (e.g., "Check-in: 20/03/2026 14:21"). The prefix text is configurable by your admin (default: "Check-in"). If the Event already has a description, the check-in note is appended on a new line.

Visit Reports are accessible from the Visit Reports tab in the RouteForce app, or from the Related Lists on the corresponding Account, Lead, or Opportunity record.

Creating Events from the Map

Click Create Event in any marker popup to schedule a new visit directly from the map. The event is pre-linked to the selected record.

11. Route History

Viewing History

Click the history icon (clock) to open the history panel. Past routes are listed with:

  • Route name (custom or auto-generated)
  • Date created
  • Step count
  • Distance and Duration (if optimized)
  • Optimized badge (green tag if the route was optimized)

Reloading a Route

Click the reload icon on any history item to restore it:

  • All stops are re-added to the route panel
  • The route line redraws on the map
  • Optimization data is preserved

Deleting History

  1. Click the trash icon on a history item
  2. A Confirm? prompt appears
  3. Click again to confirm deletion, or click the cancel icon to abort

History Map

Each saved route has a record page with an interactive mini-map (Leaflet):

  • Numbered markers show each route stop in order
  • A polyline traces the route path between stops
  • GPS check-in badges (green circles) appear on steps where the user actually checked in, showing the real GPS position
  • Color coding: green markers for checked-in steps, default color for others

To view the map, click the route name in the History list view or in the Route History panel.

12. Address Enrichment

When viewing an Account with missing or incomplete address coordinates, the Address Enrichment Panel appears in the map interface. This feature uses the French BAN API (api-adresse.data.gouv.fr) to search and auto-fill addresses:

  1. Type an address in the search field
  2. Select from the autocomplete suggestions
  3. The address fields (street, city, postal code, country) and GPS coordinates are populated automatically

Note: This feature is currently available for French addresses only. It helps ensure all your Account records are properly geocoded for map display.

13. Contact Lookup

When enabled, each route step can have a contact assigned:

  1. After adding a stop, a Contact dropdown appears on the step
  2. Search for contacts linked to the record
  3. Select a contact — they appear in the CSV export and calendar events

This is useful for knowing exactly who you'll meet at each stop.

14. Mobile Usage

RouteForce is fully responsive and works on Salesforce Mobile:

  • Filter panel slides out from the left (tap to open/close)
  • Route panel takes full width on small screens
  • Check-in modal adapts to virtual keyboard (form stays visible)
  • Map supports touch gestures (pinch to zoom, one-finger drag to pan)
  • All buttons are touch-friendly with appropriate sizing

15. Multilingual Support

RouteForce is available in English, French, and Spanish. The interface language follows your Salesforce user language setting.

To change language: Settings > Language & Time Zone > select your language. After changing your language in Salesforce, perform a hard refresh (Ctrl+Shift+R on Windows, Cmd+Shift+R on Mac) to reload all labels.

17. Custom Actions

Custom Actions allow your admin to add custom buttons on marker popups, event popups, and the daily agenda panel. These buttons launch Screen Flows in a fullscreen modal, enabling specialized workflows without leaving the map.

How It Works

  1. Custom action buttons appear in marker popups, event popups, and daily agenda rows
  2. Click a custom action button — a fullscreen modal opens with the associated Screen Flow
  3. Complete the flow — on save or cancel, the modal closes automatically

Where Custom Actions Appear

Location Description
Marker Popups Buttons on Account, Lead, and Opportunity marker popups (map click)
Event Popups Buttons on daily event marker popups (calendar pins on the map)
Daily Agenda Panel Icon buttons on each event row in the "Today's Agenda" side panel
Group Popups Buttons on each record when multiple records share an address

Display Context

The Display Context setting on each custom action controls where it appears:

Value Behavior
All (default) Button appears everywhere: popups, event popups, and agenda panel
Popup Button appears only on marker and event popups (map clicks)
EventPanel Button appears only on daily agenda panel rows

Common Use Cases

Use Case Description
Data entry forms Enter field data (e.g., product orders, inventory counts) directly from the map
Survey completion Fill out visit surveys linked to the Account
Quick updates Update record fields via a guided flow

Custom Action Appearance

Each custom action button has:

  • Label — Action name (e.g., "DN Entry", "Complete Survey")
  • Icon — Visual indicator (e.g., edit, clipboard, check)
  • Color — Configurable button color for quick identification

Conditional Visibility

Your admin can configure custom actions to appear only on specific records. For example, a "DN Entry" button might only appear on Accounts of type "Customer - Direct". If a custom action doesn't appear on a marker popup, it's because the record doesn't match the configured filter condition.

Notes

  • Custom actions are configured by your admin via RouteForce Action metadata records
  • The flow receives the record ID of the clicked marker automatically
  • Custom actions appear on both single marker popups and group popup items
  • The modal is responsive — it adapts to mobile screens and landscape orientation
  • Actions can be restricted to specific display contexts (popups only, agenda panel only, or everywhere)

18. Application Logs

RouteForce automatically records application events in a structured log. This helps your admin monitor the application health and troubleshoot issues.

What You See

As a standard user, you can view your own log entries from the RouteForce Logs tab:

Column Description
Level INFO (normal), WARN (potential issue), ERROR (failure)
Context Which part of the app generated the entry
Message Details about the event
Timestamp When it happened

Automatic Cleanup

Log entries older than 30 days are automatically deleted. You don't need to do anything — the system handles this in the background.

When to Check Logs

  • After a route optimization fails — look for ERROR entries in context "RouteService"
  • After a check-in problem — look for entries in context "EventMapService"
  • When your admin asks you to reproduce an issue — the logs capture the technical details

19. Usage Analytics Dashboard (Admin)

The Usage Analytics Dashboard is an admin-only tab that provides insights into how RouteForce is being used across your organization. It is accessible to users with the RouteForce_Admin permission set.

Automation Jobs

At the top of the dashboard, the Automation Jobs section displays the status of RouteForce's background scheduled jobs. These jobs run automatically to keep the application healthy:

Job Purpose
Usage Aggregation Summarizes daily usage data for dashboard reporting
Log Purge Deletes application log entries older than 30 days
Usage Purge Deletes raw usage records older than the retention period

For each job, the table shows:

Column Description
Status Color-coded badge indicating the job's current state
Scheduled By The user who scheduled the job
Next Run The date and time of the next scheduled execution

Job Status Badges

Badge Meaning
Active (green) The job is scheduled and running normally
Inactive (orange) The job exists but is not currently in a running state
Missing (gray) The job is not scheduled at all — it needs to be rescheduled

Rescheduling Jobs

Click the Reschedule Jobs button to manually re-schedule all three background jobs under your user account. This is useful in the following situations:

  • After a sandbox refresh — scheduled jobs are lost during sandbox refreshes and need to be re-created
  • Deactivated user — if the user who originally scheduled the jobs has been deactivated and the automatic self-rescheduling did not trigger
  • Missing or Inactive jobs — if any job shows a "Missing" or "Inactive" badge, rescheduling restores them to active status

After clicking Reschedule Jobs, the table refreshes and all three jobs should display a green Active badge with your name in the "Scheduled By" column.

"Licence expired" overlay Contact support (email shown on the overlay)

Check-in button missing Events feature or Visit Reports must be enabled by your admin

Support

For questions or issues, contact the support email configured by your admin (visible in the Configuration section of the filter panel), or reach out to your Salesforce administrator.
