Configuration Guide

A complete reference for Salesforce administrators to configure, customize, and maintain RouteForce.

1. Configuration Architecture

RouteForce uses a CMDT-first approach. All configurable settings are stored in Custom Metadata Types, which are deployable across environments and editable without code changes.

RF_Onboarding Flow (Screen Flow)
 └─► SecureConfigService ──► InternalConfig__c (API keys, protected)

RouteForceConfig__mdt (app settings, edited in Setup)
 └─► ConfigService.cls (Apex reader)
      └─► LWC Components (@wire)

3. API Key Management

API keys are stored in InternalConfig__c (Hierarchy Custom Setting), protected by the routeforce namespace — subscribers cannot read them.

KeyPurposeStored In
Licence KeyRouteForce licence validationInternalConfig__c.LicenceKey__c
ORS API KeyVPS routing server (ors.routeforce.app)InternalConfig__c.OrsApiKey__c
Fallback API KeyHeiGIT public API (fallback routing)InternalConfig__c.FallbackApiKey__c

Keys are set during the RF_Onboarding wizard and stored via SecureConfigService.storeKeys().

4. Licence System

The licence is validated via a callout to https://ors.routeforce.app/api/licence/{key}. The result is cached for 24 hours. If the server is unreachable, the app continues for 7 days (fail-open grace period).

StateBehavior
ActiveNormal operation. Green indicator in Configuration section
InactiveLicence expired or invalid. Red indicator, app blocked
Not ConfiguredNo licence key entered. Setup prompt displayed
Server UnreachableFail-open: app works for 7 days. Warning indicator

HTTP Response Handling

CodeMeaningAction
200SuccessUpdate cache, allow/block based on valid field
401Invalid keyBlock immediately (no fail-open)
403Kill switchBlock immediately (nginx-level block)
5xx / TimeoutServer error / unreachableFail-open for 7 days

5. FlexiPage Configuration

Configure via Setup > Lightning App Builder > RouteForce

Filter Fields

Define which picklist/boolean fields appear in the filter panel using a comma-separated filterFields property. Each field must be prefixed with its object name.

SyntaxExampleDescription
Object-prefixedAccount.IndustryAccount field
Object-prefixedOpportunity.StageNameOpportunity field
Object-prefixedLead.StatusLead field
Cross-objectOpportunity.Account.IndustryFilter Opportunities by their Account's field value
CMDT Fallback: If the FlexiPage filterFields property is empty, the system reads the default from RouteForceConfig__mdt.FilterFields__c. The FlexiPage value always takes priority.

7. Query Filter Security

Admin-provided SOQL filters (ContactQueryFilter__c, LeadQueryFilter__c, OpportunityQueryFilter__c) are validated using an allowlist regex before being injected into queries.

  • Allowed: field comparisons (=, !=, <, >, <=, >=), LIKE, IN, NOT IN, INCLUDES, EXCLUDES, AND, OR, parentheses, date literals
  • Rejected: subqueries, DML keywords, semicolons — anything that doesn't match the pattern

If a filter is rejected, the system falls back to the default filter and logs a warning in RouteForceLog__c.
