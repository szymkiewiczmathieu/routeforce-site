RouteForce Configuration Guide

A complete reference for Salesforce administrators to configure, customize, and maintain RouteForce.

1. Configuration Architecture

RouteForce uses a CMDT-first approach. All configurable settings are stored in Custom Metadata Types, which are:

  • Deployable across environments
  • Editable without code changes
  • Accessible to the onboarding wizard (Screen Flow)

2. RouteForceConfig__mdt — Application Settings

The central configuration record. All settings are in the Default record.

Map Settings

Field Type Default Description
AddressFieldPrefix__c Picklist Shipping Which compound address to use for geolocation: Shipping (ShippingLatitude/Longitude) or Billing (BillingLatitude/Longitude)
DefaultMapLayer__c Picklist Standard Initial basemap: Standard (OpenStreetMap), Terrain, or Satellite
DefaultZoom__c Number 10 Initial zoom level (1 = world, 18 = street level)
DefaultDisplayMode__c Text both Default marker display: accounts, leads, or both
ClusterRadius__c Number 60 Marker clustering radius in pixels. Higher = more aggressive clustering
DefaultNearbyRadius__c Number 10 Default GPS nearby search radius in kilometers

Feature Toggles

Field Type Default Description
EnableEvents__c Checkbox true Calendar events integration (agenda panel, event markers, check-in)
EnableLeads__c Checkbox false Display geocoded Leads as markers on the map
EnableOpportunities__c Checkbox false Display Opportunities (via parent Account address) as markers
EnableNearby__c Checkbox true GPS "Nearby" search button in toolbar
EnableVisitReports__c Checkbox true Create VisitReport__c records during Event check-in
EnableCSVExport__c Checkbox true Show CSV export button after route optimization
EnableHistoryNaming__c Checkbox true Allow users to name saved routes (prompt before saving)
QuickTextEnabled__c Checkbox false Quick Text snippet insertion in check-in visit report modal

Route Settings

Field Type Default Description
MaxRouteSteps__c Number 50 Maximum stops per route (ORS API limit is 50)
EventOppLinkTarget__c Text opportunity When creating an Event from an Opportunity marker: opportunity (link Event to Opp) or account (link Event to parent Account)

Query Configuration

Field Type Default Description

ContactQueryFields__c Text Id,Name,Email,Phone Comma-separated Contact fields to display in popups
ContactQueryFilter__c Text (empty) Optional SOQL WHERE clause for Contacts (e.g., IsActive = true)

OpportunityQueryFields__c Text (empty) Additional Opportunity fields (e.g., NextStep,Probability)
OpportunityQueryFilter__c Text IsClosed = false SOQL WHERE for Opportunity queries. Default shows open Opps only
LastVisitField__c Text (empty) Account field API name for "last visit" date. Enables the "Not visited since" filter
NotVisitedSinceDays__c Number 30 Default threshold in days for the "Not visited since" filter

System

Field Type Default Description

3. API Key Management

API keys are stored in InternalConfig__c (Hierarchy Custom Setting), protected by the routeforce namespace — subscribers cannot read them.

Key Purpose Stored In
Licence Key RouteForce licence validation InternalConfig__c.LicenceKey__c
ORS API Key VPS routing server (ors.routeforce.app) InternalConfig__c.OrsApiKey__c
Fallback API Key HeiGIT public API (fallback routing) InternalConfig__c.FallbackApiKey__c

These keys are set during the RF_Onboarding wizard (Step 1 — API Keys) and stored via SecureConfigService.storeKeys().

4. Licence System

How It Works

  1. Validation callout → https://ors.routeforce.app/api/licence/{key}
  2. Server returns: { valid: true, expires: "2026-12-31", client: "Acme Corp" }
  3. Result is cached in LicenceCache__c for 24 hours
  4. If the server is unreachable, the app continues for 7 days (fail-open grace period)
  5. After 7 days without contact, the app blocks with a licence error overlay

Licence States

State Behavior
Active Normal operation. Green indicator in Configuration section
Inactive Licence expired or invalid. Red indicator, app blocked
Not Configured No licence key entered. Setup prompt displayed
Server Unreachable Fail-open: app works for 7 days. Warning indicator

Cache Details

Field Description
LicenceCache__c.IsValid__c true if last validation succeeded
LicenceCache__c.CheckedAt__c Timestamp of last callout
LicenceCache__c.ClientName__c Client name from server response
LicenceCache__c.ExpiresAt__c Licence expiration date

HTTP Response Handling

Code Meaning Action
200 Success Update cache, allow/block based on valid field
401 Invalid key Block immediately (no fail-open)
403 Kill switch Block immediately (nginx-level block)
5xx Server error Fail-open for 7 days
Timeout Unreachable Fail-open for 7 days

5. FlexiPage Configuration

The RouteForce FlexiPage is the main map interface. Configure it via Setup > Lightning App Builder > RouteForce.

Filter Fields (filterField1–16)

Define which picklist/boolean fields appear in the filter panel. Supports three syntaxes:

Syntax Example Description
Simple Industry Account field (no prefix needed)
Object-prefixed Opportunity.StageName Field on a specific object
Cross-object Opportunity.Account.Industry Filter Opportunities by their Account's field value

Supported field types: Picklist, Multi-Picklist, Boolean (Checkbox)

Default configuration (out of the box):

Slot Value Description
filterField1 Account.Industry Account Industry
filterField2 Account.Type Account Type
filterField3 Account.Rating Account Rating
filterField4 Account.Ownership Account Ownership
filterField5 Opportunity.StageName Opportunity Stage
filterField6 Opportunity.ForecastCategory Opportunity Forecast
filterField7 Lead.Status Lead Status
filterField8 Lead.LeadSource Lead Source
filterField9 Opportunity.Account.Industry Cross-object: Opp by Account Industry
filterField10 Lead.Rating Lead Rating
filterField11–16 (empty) Available for custom filters

Legend Configuration

The legend assigns distinct colors to marker values. Configure these comma-separated lists:

Property Example Description
typeField Type Account field used for marker color assignment
typeValues Customer - Direct,Customer - Channel,Prospect,Other Account legend entries (blue tones)
oppTypeValues Prospecting,Qualification,Needs Analysis,Value Proposition,Negotiation/Review,Closed Won,Closed Lost Opportunity legend entries (warm tones)
leadTypeValues Open - Not Contacted,Working - Contacted,Closed - Converted,Closed - Not Converted Lead legend entries (cool tones)

Order matters — the first value gets the first color in the palette, and so on.

Marker Popup Fields

Configure which fields appear in marker popups (any field type supported):

Phone,Industry,Website,Opportunity.StageName,Opportunity.Amount,Opportunity.CloseDate,Lead.Company,Lead.LeadSource,Lead.Status,Lead.Email
Syntax Example
Account field Phone, Industry, Website
Opportunity field Opportunity.Amount, Opportunity.NextStep
Lead field Lead.Company, Lead.Email

Other Properties

Property Type Default Description
addressFieldPrefix String Shipping Overrides CMDT for this FlexiPage instance
showRecordType Boolean false Adds Record Type filter dropdown
forceMultiPicklist String Industry,Type Comma-separated fields rendered as multi-select checkboxes

7. Data Requirements

Account Geocoding

Accounts must have latitude/longitude fields populated to appear on the map:

Address Prefix Latitude Field Longitude Field
Shipping ShippingLatitude ShippingLongitude
Billing BillingLatitude BillingLongitude

How to geocode: Salesforce provides automatic geocoding via Data.com Clean (requires licence) or you can use third-party tools (Geocodio, Google Geocoding API) to batch-update coordinates.

Tip: Records without coordinates are silently excluded from the map — no error is thrown.

Lead Geocoding

Leads use their standard Latitude and Longitude fields. These are part of the Lead Address compound field.

Opportunity Geocoding

Opportunities do not have their own address. RouteForce uses the parent Account's address (via the AccountId relationship). The Account must be geocoded.

Events

Events appear on the map when they have a related record (WhatId or WhoId) that has geocoded coordinates. Events without a geocoded related record show in the calendar but not on the map.

8. Remote Site Settings

RouteForce requires callout access to two external services:

Name URL Purpose
RouteForce_ORS https://ors.routeforce.app Route optimization, directions, licence validation
RouteForce_HeiGIT https://api.openrouteservice.org Fallback routing (when VPS is unavailable)

These are included in the managed package and deployed automatically on install. If missing, add them manually:

  1. Setup > Remote Site Settings > New
  2. Enter the name and URL
  3. Check Active
  4. Save

11. Adding Custom Filter Fields

Example: Add "Account.NumberOfEmployees" as a filter

  1. Open Setup > Lightning App Builder
  2. Edit the RouteForce AppPage
  3. Click the RouteForce component
  4. Find an empty filter slot (e.g., filterField11)
  5. Enter: Account.NumberOfEmployees
  6. Save and activate

Note: Only Picklist, Multi-Picklist, and Boolean fields are supported as filters. Text and Number fields are not filterable (but can be shown in marker popups via markerFields).

Example: Add a cross-object filter

To filter Opportunities by their parent Account's Rating:

  1. Find an empty filter slot
  2. Enter: Opportunity.Account.Rating
  3. Save

This adds a Rating filter that, when selected, only shows Opportunities whose parent Account matches the selected Rating value.

Example: Add a field to marker popups

To show Opportunity Probability in popups:

  1. Find the markerFields property
  2. Add Opportunity.Probability to the comma-separated list
  3. Save

12. Customizing the Legend

Adding a new Account type value

If your org has a custom Type value "Strategic Partner":

  1. Edit the FlexiPage
  2. Find typeValues
  3. Add it: Customer - Direct,Customer - Channel,Prospect,Other,Strategic Partner
  4. Save — the new value gets the next color in the blue palette

Changing Opportunity legend values

If your org uses custom Opportunity stages:

  1. Find oppTypeValues
  2. Replace with your stages: Discovery,Proposal,Negotiation,Closed Won,Closed Lost
  3. Save

Color Assignment

Colors are assigned by position in the comma-separated list:

Position Account Color Opportunity Color Lead Color

4th Lighter Blue Yellow Green

13. Multi-Select Filter Configuration

By default, picklist filters render as single-select dropdowns. To render them as multi-select checkbox lists (allowing multiple selections):

  1. Edit the FlexiPage
  2. Find the forceMultiPicklist property
  3. Enter comma-separated field API names: Industry,Type,Rating
  4. Save

Only fields already configured in filterField1–16 are affected.

14. Record Type Filter

To add a Record Type dropdown to the filter panel:

  1. Edit the FlexiPage
  2. Set showRecordType to true
  3. Save

This adds a dropdown listing all active Account Record Types. Users can filter the map by Record Type.

15. Routing Architecture

Primary: VPS ORS

  • URL: https://ors.routeforce.app
  • Auth: API key in X-API-Key header
  • Services: VROOM optimization (optimal stop order) + Directions (route geometry)
  • Limit: 50 waypoints per request

Fallback: HeiGIT Public API

  • URL: https://api.openrouteservice.org
  • Auth: API key in query parameter
  • Used when: VPS returns 5xx or times out
  • Limit: Lower rate limits (public tier)

Kill Switch

The VPS nginx can disable the X-API-Key globally, returning 401 for all requests. This blocks both routing and licence validation simultaneously.

Error Handling

Error User Impact
400 (Bad Request) "Invalid coordinates" — point is not routable (ocean, no road data)
401 (Unauthorized) "API key invalid" — check InternalConfig__c keys
5xx (Server Error) Automatic fallback to HeiGIT. If both fail, error toast
Timeout Automatic retry (3 attempts), then fallback

17. Troubleshooting Configuration Issues

Issue Diagnostic Solution
Changes not taking effect CMDT deployment is async Wait 10–30 seconds, then refresh
"Licence not configured" No key in InternalConfig__c Run RF_Onboarding, enter licence key
Leads/Opps not showing Feature toggle off Enable in RF_Onboarding or edit CMDT directly
Filter field not appearing Wrong syntax Check: ObjectType.FieldName format, field must be Picklist/Boolean
Cross-object filter empty No matching records Verify records exist with both the filter value and geocoded address
Legend colors wrong Order mismatch Reorder typeValues — position determines color
"Not visited since" missing LastVisitField__c empty Set to your Account date field API name
Contact lookup empty No related contacts Ensure Contacts exist on the Account
Events not on map Missing geocoded related record The Event's related Account/Lead must have coordinates
Quick Text not showing Feature off Enable QuickTextEnabled__c in CMDT
