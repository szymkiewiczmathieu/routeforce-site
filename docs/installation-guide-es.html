2. Instalación del paquete

Paso 2.1 — Instalar mediante enlace

Haga clic en el enlace de su entorno:

EntornoEnlace
Production / Developer Instalar en producción
Sandbox Instalar en sandbox

Paso 2.2 — Opciones de instalación

  1. Inicie sesión con un perfil System Administrator
  2. Seleccione "Install for Admins Only" (recomendado para la configuración inicial)
  3. Marque "Yes, grant access to these third-party web sites" si se solicita
  4. Haga clic en Install
  5. Espere el email de confirmación (1 a 5 minutos)
⚠️ Namespace : El paquete utiliza el namespace roteforce. Todos los componentes, objetos y campos se prefijan automáticamente (ej.: roteforce__RoteForceConfig__mdt).

3. Remote Site Settings & CSP

El componente realiza llamadas HTTP al servidor de optimización y carga tiles de mapa desde OpenStreetMap. Se requieren dos configuraciones:

Paso 3.1 — Remote Site Settings

Setup → Security → Remote Site Settings → New

NameURLDescription
RoteForce_ORShttps://ors.roteforce.appServidor de optimización y enrutamiento (obligatorio)

Paso 3.2 — CSP Trusted Sites

Setup → Security → CSP Trusted Sites → New

NameURLContext
RoteForce_ORShttps://ors.roteforce.appAll (connect-src)
OpenStreetMap_Tileshttps://*.tile.openstreetmap.orgAll (img-src)
Leaflet_CDNhttps://unpkg.comAll (script-src, style-src)
Esri_Satellitehttps://server.arcgisonline.comAll (img-src, connect-src) — requerido para el fondo Satellite
OpenTopoMaphttps://*.opentopomap.orgAll (img-src) — requerido para el fondo Terrain

Estos sitios son necesarios para cargar el mapa, los tiles y los scripts Leaflet.js.

5. Configuración de la licencia

RoteForce utiliza un sistema de licencia para activar el componente. Las claves son proporcionadas por soporte tras el registro (prueba de 14 días o suscripción).

Paso 5.1 — Configurar la licencia

Acceda al Custom Metadata Type LicenceConfig :

Setup → Custom Metadata Types → "Licence Config" → Manage Records → Edit "Default"
CampoValorDescription
Licence Key (proporcionada por soporte) Obligatorio Clave de licencia RoteForce. Sin esta clave, el componente muestra una pantalla de licencia expirada.
ORS API Key (proporcionada por soporte) Obligatorio Clave de autenticación para el servidor de optimización. Sin esta clave, los cálculos de ruta y la optimización no funcionarán.
💡 Prueba gratuita: Contacte contact@roteforce.app para obtener una prueba gratuita de 14 días. Recibirá su clave de licencia y su clave API por email seguro (enlace único, auto-destrucción tras lectura).
💡 Activación de la licencia: La validación de licencia es automática. Una vez ingresada la clave, RoteForce verifica la validez con nuestros servidores y activa la aplicación. Al cambiar de clave, la actualización se efectúa en 24 horas.

6. Configuración Custom Metadata

Toda la configuración de RoteForce está centralizada en un solo Custom Metadata Type: RoteForceConfig. Un registro Default se proporciona con valores predeterminados. Todos los campos son SubscriberControlled : puede modificarlos libremente.

Setup → Custom Metadata Types → "RoteForce Config" → Manage Records → Edit "Default"

6.2 — Configuración por página (override App Builder)

Las propiedades configurables en el App Builder (paso 7) son prioritarias sobre los Custom Metadata. Esto permite tener configuraciones diferentes por página (ej.: Billing en la página Accont, Shipping en una página dedicada).

7. Configuración App Builder

Paso 7.1 — Agregar el componente a una página Lightning

  1. Abra una página Lightning en el App Builder (página Accont, Home o App Page)
  2. Busque "roteforce" en el panel de componentes (a la izquierda)
  3. Arrastre y suelte el componente en la página
  4. Configure las propiedades en el panel derecho (ver abajo)
  5. Haga clic en Save luego Activate

7.2 — Propiedades configurables

💡 Filtros dinámicos: Les filtres s'adaptent automatiquement au type de champ (picklist → dropdown, booléen → toggle, multipicklist → sélection multiple). Vos povez filtrer sur n'importe quel champ standard o custom de l'objet Accont.

Étape 7.3 — Ajoter l'onglet RoteForce à la barre de navigation

Recomendado La pestaña RoteForce donne accès à la carte complète (vue tos comptes). Ajotez-le à votre application Lightning :

  1. Vaya a Setup → App Manager
  2. Haga clic en Edit en su aplicación (ej.: Sales)
  3. Onglet Navigation Items
  4. Ajotez RoteForce a los elementos seleccionados
  5. Positionnez-le à l'emplacement sohaité et cliquez Save

Étape 7.4 — Ajoter l'onglet Map sur la fiche Accont

Recomendado Por voir la carte centrée sur un compte spécifique, ajotez un onglet dans la page Accont :

  1. Ouvrez la page Accont dans l'App Builder
  2. Ajotez un composant Tabs si absent, o utilisez les onglets existants
  3. Créez un novel onglet nommé "Map" (o "Carte")
  4. Arrastre y suelte el componente RoteForce en esta pestaña
  5. Haga clic en Save luego Activate

💡 Le composant se centre automatiquement sur le compte corant lorsqu'il est positionné sur une page Accont. Les comptes à proximité sont visibles sur la carte, facilitant la planification de visites gropées.

9. Buenas prácticas

Recomendación: campo Rollup "Última visita" Recomendado

Por exploiter pleinement le filtre "Non visité deluego X jors" et identifier les comptes à revisiter, nos recommandons encarecidamente de créer un champ Rollup Summary sur l'objet Accont :

Opción A — Rollup Summary (recomendado)

ParámetroValor
LabelÚltima Visita (o Last Visit Date)
API NameLast_Visit_Date__c
TypeRollup Summary
Summarized ObjectEvents (activités liées au compte)
Roll-Up TypeMAX
Field to AggregateActivityDate (Fecha del evento)
Filter CriteriaSubject CONTAINS 'Visite' — o filtrez par RecordType dédié si vos en avez un

Opción B — Flow programado (alternativa)

Si el Rollup Summary no es posible (ej.: filtros complejos, objetos no-master-detail), cree un Scheduled Flow diario que:

  1. Requête les Events des dernières 24h liés à des Acconts
  2. Met à jor Last_Visit_Date__c sur chaque Accont concerné

Configuration dans RoteForce

Ingrese el nombre API del campo en RoteForce Config → LastVisitField__c (ex: Last_Visit_Date__c). Le filtre "Non visité deluego" sera alors disponible dans la carte.

Caso de uso: identificar cuentas sin visita reciente

Avec ce champ en place, vos povez :

  • 📍 Dans RoteForce : utiliser le filtre "Non visité deluego X jors" por afficher uniquement les comptes à revisiter sur la carte
  • 📊 Dans un rapport/dashboard SF : crear un informe sobre las cuentas donde Last_Visit_Date__c < TODAY() - 30
  • 🔍 En SOQL (por les admins) :
SELECT Name, BillingCity, Last_Visit_Date__c, OwnerId
FROM Accont
WHERE Last_Visit_Date__c < LAST_N_DAYS:30
   OR Last_Visit_Date__c = NULL
ORDER BY Last_Visit_Date__c ASC NULLS FIRST

↑ Cette requête retorne tos les comptes non visités deluego 30 jors (o jamais visités). Idéal por alimenter une vue liste o un dashboard de suivi commercial.

💡 Porquoi un Rollup plutôt que LastActivityDate ? El campo estándar LastActivityDate inclut les appels, emails, tâches et tos types d'activités — pas uniquement les visites terrain. Un Rollup dédié (o Flow) vos permet de filtrer precisamente sur les visites physiques et d'identifier les comptes qui n'ont pas reçu de visite deluego N jors. C'est la base d'une bonne coverture terrain.

Recomendación: fondos de mapa alternativos

Por activer les fonds Satellite (imágenes aéreas Esri) y Terrain (OpenTopoMap), ajotez les CSP Trusted Sites correspondantes (voir étape 3.2). Sans ces sites autorisés, seul le fond Standard (OpenStreetMap) fonctionnera.

10. Solución de problemas

ProblemaSolution
Pantalla "🔒 Licencia expirada" Verifique que Licence Config → Default contiene una clave de licencia válida. Contacte contact@roteforce.app por un trial.
Sin marcadores en el mapa Verifique que vos comptes ont des coordonnées GPS (Setup → Data Integration Rules → Geocodes activés). Vérifiez le Address Field Prefix en el CMDT.
Erreur "Optimiser la tornée" Verifique que l'ORS API Key está configurada en Licence Config y que el Remote Site Setting https://ors.roteforce.app está activo.
Componente no visible en el App Builder Verifique que le Permission Set RoteForce User o RoteForce Admin está asignado a su usuario.
Mapa en blanco (sin tiles) Ajotez les CSP Trusted Sites por https://*.tile.openstreetmap.org et https://unpkg.com (paso 3).
Fond Satellite o Terrain ne charge pas Ajotez les CSP Trusted Sites : https://server.arcgisonline.com (Satellite) et https://*.opentopomap.org (Terrain). Ver paso 3.2.
Erreur 401 sur les rotes Votre clé API est invalide o votre licence a expiré. Contacte le support.
Los filtros no muestran nada Verifique que les Filter Fields configurés dans l'App Builder correspondent à des champs existants sur l'objet Accont.
