1. Requisitos previos

ElementoRequerido

2. Instalación del paquete

Paso 2.1 — Instalar mediante enlace

Haga clic en el enlace correspondiente a su entorno:

EntornoEnlace
Production / Developer Instalar en producción
Sandbox Instalar en sandbox

Paso 2.2 — Opciones de instalación

  1. Inicie sesión con un perfil de System Administrator
  2. Seleccione "Install for Admins Only" (recomendado para la configuración inicial)
  3. Marque "Yes, grant access to these third-party web sites" si se solicita
  4. Haga clic en Install
  5. Espere el email de confirmación (1 a 5 minutos)
⚠️ Namespace : El paquete utiliza el namespace routeforce. Todos los componentes, objetos y campos se prefijan automáticamente (ex : routeforce__RouteForceConfig__mdt).

3. Remote Site Settings & CSP

El componente realiza llamadas HTTP al servidor de optimización y carga mosaicos de mapa desde OpenStreetMap. Se requieren dos configuraciones:

Paso 3.1 — Remote Site Settings

Setup → Security → Remote Site Settings → New

NameURLDescription
RouteForce_ORShttps://ors.routeforce.appServidor de optimización y enrutamiento (obligatorio)

Paso 3.2 — CSP Trusted Sites

Setup → Security → CSP Trusted Sites → New

NameURLContext
RouteForce_ORShttps://ors.routeforce.appAll (connect-src)
OpenStreetMap_Tileshttps://*.tile.openstreetmap.orgAll (img-src)
Leaflet_CDNhttps://unpkg.comAll (script-src, style-src)
Esri_Satellitehttps://server.arcgisonline.comAll (img-src, connect-src) — requerido para el fondo Satélite
OpenTopoMaphttps://*.opentopomap.orgAll (img-src) — requerido para el fondo Terreno

Estos sitios son necesarios para cargar el mapa, los mosaicos y los scripts de Leaflet.js.

5. Configuración de la licencia

RouteForce utiliza un sistema de licencia para activar el componente. Las claves son proporcionadas por soporte después del registro (prueba de 14 días o suscripción).

Paso 5.1 — Introducir la licencia

Acceda al Custom Metadata Type LicenceConfig:

Setup → Custom Metadata Types → "Licence Config" → Manage Records → Edit "Default"
CampoValorDescripción
Licence Key (proporcionada por soporte) Obligatoire Clave de licencia RouteForce. Sin esta clave, el componente muestra una pantalla de licencia expirada.
ORS API Key (proporcionada por soporte) Obligatoire Clave de autenticación para el servidor de optimización. Sin esta clave, los cálculos de ruta y la optimización no funcionarán.
💡 Prueba gratuita : Contacte contact@routeforce.app para una prueba gratuita de 14 días. Recibirá su clave de licencia y su clave API por email seguro (enlace de un solo uso, auto-destruido después de la lectura).
💡 Activación de la licencia : La validación de licencia es automática. Una vez introducida la clave, RouteForce verifica la validez con nuestros servidores y activa la aplicación. Si la clave cambia, la actualización se hace efectiva en 24h.

6. Configuración de Custom Metadata

Toda la configuración de RouteForce se centraliza en un único Custom Metadata Type : RouteForceConfig. Se proporciona un registro Default con valores predeterminados. Todos los campos son SubscriberControlled: puede modificarlos libremente.

Setup → Custom Metadata Types → "RouteForce Config" → Manage Records → Edit "Default"

6.1 — Campos de configuración

CampoTipoPredeterminadoDescripción
Address Field Prefix Picklist Shipping Optionnel Prefijo de dirección para geocodificación : Shipping ou Billing. Fallback automático al otro tipo si la dirección preferida no tiene coordenadas.
Enable Contacte Lookup Checkbox true Optionnel Activa la visualización de contactos en los popups de marcadores y la selección de contactos al planificar una ruta.
Contacte Query Fields Text Id,Name Optionnel Contacte fields to retrieve. Examples : Id,Name,Email,Phone,Title
Contacte Query Filter Text (vide) Optionnel Cláusula SOQL WHERE (sin la palabra clave WHERE). Ej. : IsActive__c = true
Default Map Layer Picklist Standard Optionnel Fondo de mapa predeterminado : Standard (calles), Terrain (topo), Satellite (vista aérea).
Enable History Naming Checkbox true Optionnel Solicita un nombre de ruta antes de guardar en el historial.
Última Visita Field Text (vide) Optionnel Nombre API del campo Account que almacena la fecha de última visita. Activa el filtro "No visitado desde". Ej. : LastActivityDate ou Derniere_Visite__c
Not Visited Since (Days) Number (vide) Optionnel Número predeterminado de días para el filtro "No visitado desde".
Support Email Email (vide) Optionnel Email mostrado en la pantalla de licencia expirada. Si está vacío, contact@routeforce.app se utiliza.
💡 Configuración mínima : Para una configuración rápida, solo las claves en Licence Config (paso 5) son obligatorias. Todos los campos anteriores tienen valores predeterminados adecuados. Puede ajustarlos después de las primeras pruebas.

6.2 — Configuración por página (override App Builder)

Las propiedades configurables en el App Builder (paso 7) tienen prioridad sobre los Custom Metadata. Esto permite diferentes configuraciones por página (ex : Billing en la página Account, Shipping en una página dedicada).

7. Configuración de App Builder

Paso 7.1 — Añadir el componente a una página Lightning

  1. Abra una página Lightning en el App Builder (página Account, Home o App Page)
  2. Busque "routeforce" en el panel de componentes (lado izquierdo)
  3. Arrastre y suelte el componente en la página
  4. Configure las propiedades en el panel derecho (ver más abajo)
  5. Haga clic en Save y luego en Activate

7.2 — Propiedades configurables

PropiedadDescripciónEjemplo
Address Field Prefix Override del CMDT: prefijo de dirección Shipping ou Billing
Show Record Type Mostrar filtro por tipo de registro Account true / false
Filter Field 1–12 Campos Account a exponer como filtros dinámicos (hasta 12) Industry, Rating, Type
Type Field Campo utilizado para colorear marcadores por categoría Type
Type Values Valores del campo tipo (separados por coma) Customer - Direct,Prospect,Partner
💡 Filtros dinámicos : Los filtros se adaptan automáticamente al tipo de campo (picklist → dropdown, booleano → toggle, multipicklist → selección múltiple). Puede filtrar por cualquier campo estándar o personalizado del objeto Account.

Paso 7.3 — Añadir la pestaña RouteForce a la barra de navegación

Recommandé La pestaña RouteForce da acceso al mapa completo (vista de todas las cuentas). Añádala a su aplicación Lightning:

  1. Vaya a Setup → App Manager
  2. Haga clic en Edit en su aplicación (ej.: Sales)
  3. Vaya a la pestaña Navigation Items
  4. Añada RouteForce a los elementos seleccionados
  5. Colóquelo en la posición deseada y haga clic en Save

Paso 7.4 — Añadir la pestaña Map en la página Account

Recommandé Para ver el mapa centrado en una cuenta específica, añada una pestaña en la página Account:

  1. Abra la página Account en el App Builder
  2. Añada un componente Tabs si no existe, o utilice las pestañas existentes
  3. Cree una nueva pestaña llamada "Map" (o "Mapa")
  4. Arrastre y suelte el componente RouteForce en esta pestaña
  5. Haga clic en Save y luego en Activate

💡 El componente se centra automáticamente en la cuenta actual cuando se coloca en una página Account. Las cuentas cercanas son visibles en el mapa, facilitando la planificación de visitas agrupadas.

⚡ Recommandation performances : Privilégiez le placement du composant dans un onglet "Map" sur la page Account plutôt que sur une page standalone (tab RouteForce). Sur une page Account, seuls les comptes à proximité sont chargés. Sur la page standalone, tous les comptes géolocalisés de votre org sont récupérés (potentiellement des milliers), ce qui peut ralentir le chargement initial. L'onglet Account offre des performances optimales.

  4. ✅ El componente se carga (mapa Leaflet visible, sin pantalla "licencia expirada")
  5. ✅ Las cuentas con direcciones geocodificadas aparecen como marcadores
  6. ✅ El botón "Ruta" abre el panel de planificación
  7. ✅ Hacer clic en los marcadores los añade como paradas
  8. ✅ El botón "Optimizar" reordena las paradas (requiere ORS API Key)
  9. ✅ El botón "Historial" muestra las rutas guardadas
  10. ✅ Después de "Guardar en calendario", un modal solicita nombrar la ruta (si Enable History Naming = true)
  11. ✅ El botón "Ocultar cuentas" aparece después de la optimización para una vista limpia de la ruta
  12. ✅ La pestaña RouteForce es visible en la barra de navegación
  13. ✅ La pestaña Map es visible en la página Account (si está configurada)

9. Buenas prácticas

Recomendación: campo Rollup "Última visita" Recommandé

Para aprovechar al máximo el filtro "No visitado en X días" e identificar las cuentas a revisitar, recomendamos encarecidamente crear un campo Rollup Summary en el objeto Account:

Opción A — Rollup Summary (recomendado)

ParámetroValor
LabelÚltima Visita
API NameLast_Visit_Date__c
TypeRollup Summary
Summarized ObjectEvents (actividades relacionadas con la cuenta)
Roll-Up TypeMAX
Field to AggregateActivityDate (Fecha del evento)
Filter CriteriaSubject CONTAINS 'Visite' — o filtre por RecordType dedicado si tiene uno

Opción B — Flow programado (alternativa)

Si un Rollup Summary no es posible (ej.: filtros complejos, objetos no master-detail), cree un Scheduled Flow diario que:

  1. Consulte los Events de las últimas 24h relacionados con Accounts
  2. Actualice Last_Visit_Date__c en cada Account correspondiente

Configuración en RouteForce

Introduzca el nombre API del campo en RouteForce Config → LastVisitField__c (ex: Last_Visit_Date__c). El filtro "No visitado desde" estará entonces disponible en el mapa.

Caso de uso: identificar cuentas sin visitas recientes

Con este campo configurado, puede:

  • 📍 Dans RouteForce : utiliser le filtre "No visitado en X días" pour afficher uniquement les comptes à revisiter sur la carte
  • 📊 En un informe/dashboard SF: cree un informe sobre las cuentas donde Last_Visit_Date__c < TODAY() - 30
  • 🔍 En SOQL (para admins):
SELECT Name, BillingCity, Last_Visit_Date__c, OwnerId
FROM Account
WHERE Last_Visit_Date__c < LAST_N_DAYS:30
   OR Last_Visit_Date__c = NULL
ORDER BY Last_Visit_Date__c ASC NULLS FIRST

↑ Esta consulta devuelve todas las cuentas no visitadas en los últimos 30 días (o nunca visitadas). Ideal para alimentar una vista de lista o un dashboard de seguimiento comercial.

💡 ¿Por qué un Rollup en lugar de LastActivityDate? El campo estándar LastActivityDate incluye llamadas, emails, tareas y todos los tipos de actividad — no solo visitas de campo. Un Rollup dedicado (o Flow) le permite filtrar con precisión las visitas físicas e identificar las cuentas que no han recibido una visita en N días. Es la base de una buena cobertura territorial.

Recomendación: fondos de mapa alternativos

Para activar los fondos Satélite (imágenes aéreas Esri) y Terreno (OpenTopoMap), añada los CSP Trusted Sites correspondientes (ver paso 3.2). Sin estos sitios autorizados, solo funcionará el fondo Estándar (OpenStreetMap).

10. Solución de problemas

ProblemaSolución
Pantalla "🔒 Licencia expirada" Verifique que Licence Config → Default contiene una clave de licencia válida. Contacte contact@routeforce.app para una prueba.
Sin marcadores en el mapa Verifique que sus cuentas tienen coordenadas GPS (Setup → Data Integration Rules → Geocodes activados). Compruebe el Address Field Prefix en el CMDT.
Error "Optimizar ruta" Vérifiez que l'ORS API Key está completada en Licence Config y que el Remote Site Setting https://ors.routeforce.app está activo.
Componente no visible en App Builder Verifique que el Permission Set RouteForce User o RouteForce Admin está asignado a su usuario.
Mapa en blanco (sin mosaicos) Añada CSP Trusted Sites para https://*.tile.openstreetmap.org et https://unpkg.com (paso 3).
Fondo Satélite o Terreno no carga Añada CSP Trusted Sites: https://server.arcgisonline.com (Satélite) y https://*.opentopomap.org (Terreno). Ver paso 3.2.
Error 401 en rutas Votre clé API est invalide ou votre licence a expiré. Contacte le support.
Los filtros no muestran nada Verifique que los Filter Fields configurados en App Builder corresponden a campos existentes en el objeto Account.
