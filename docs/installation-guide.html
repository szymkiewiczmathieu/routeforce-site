2. Installation du package

Étape 2.1 — Installer via le lien

Cliquez sur le lien correspondant à votre environnement :

EnvironnementLien
Production / Developer Installer en production
Sandbox Installer en sandbox

Étape 2.2 — Options d'installation

  1. Connectez-vous avec un profil System Administrator
  2. Sélectionnez "Install for Admins Only" (recommandé pour la configuration initiale)
  3. Cochez "Yes, grant access to these third-party web sites" si demandé
  4. Cliquez sur Install
  5. Attendez l'email de confirmation (1 à 5 minutes)
⚠️ Namespace : Le package utilise le namespace routeforce. Tous les composants, objets et champs sont préfixés automatiquement (ex : routeforce__RouteForceConfig__mdt).

3. Remote Site Settings & CSP

Le composant effectue des appels HTTP vers le serveur d'optimisation et charge des tuiles de carte depuis OpenStreetMap. Deux configurations sont nécessaires :

Étape 3.1 — Remote Site Settings

Setup → Security → Remote Site Settings → New

NameURLDescription
RouteForce_ORShttps://ors.routeforce.appServeur d'optimisation et routing (obligatoire)

Étape 3.2 — CSP Trusted Sites

Setup → Security → CSP Trusted Sites → New

NameURLContext
RouteForce_ORShttps://ors.routeforce.appAll (connect-src)
OpenStreetMap_Tileshttps://*.tile.openstreetmap.orgAll (img-src)
Leaflet_CDNhttps://unpkg.comAll (script-src, style-src)
Esri_Satellitehttps://server.arcgisonline.comAll (img-src, connect-src) — requis pour le fond Satellite
OpenTopoMaphttps://*.opentopomap.orgAll (img-src) — requis pour le fond Terrain

Ces sites sont nécessaires pour le chargement de la carte, des tuiles et des scripts Leaflet.js.

5. Configuration de la licence

RouteForce utilise un système de licence pour activer le composant. Les clés sont fournies par le support après inscription (trial 14 jours ou abonnement).

Étape 5.1 — Renseigner la licence

Accédez au Custom Metadata Type LicenceConfig :

Setup → Custom Metadata Types → "Licence Config" → Manage Records → Edit "Default"
ChampValeurDescription
Licence Key (fournie par le support) Obligatoire Clé de licence RouteForce. Sans cette clé, le composant affiche un écran de licence expirée.
ORS API Key (fournie par le support) Obligatoire Clé d'authentification pour le serveur d'optimisation. Sans cette clé, les calculs de route et l'optimisation ne fonctionneront pas.
💡 Trial gratuit : Contactez contact@routeforce.app pour obtenir un essai gratuit de 14 jours. Vous recevrez votre clé de licence et votre clé API par email sécurisé (lien unique, auto-destruction après lecture).
💡 Activation de la licence : La validation de licence est automatique. Une fois la clé renseignée, RouteForce vérifie la validité auprès de nos serveurs et active l'application. En cas de changement de clé, la mise à jour est effective sous 24h.

6. Configuration Custom Metadata

Toute la configuration de RouteForce est centralisée dans un seul Custom Metadata Type : RouteForceConfig. Un enregistrement Default est fourni avec des valeurs par défaut. Tous les champs sont SubscriberControlled : vous pouvez les modifier librement.

Setup → Custom Metadata Types → "RouteForce Config" → Manage Records → Edit "Default"

6.1 — Champs de configuration

ChampTypeDéfautDescription
Address Field Prefix Picklist Shipping Optionnel Préfixe d'adresse pour la géolocalisation : Shipping ou Billing. Fallback automatique sur l'autre type si l'adresse préférée n'a pas de coordonnées.
Enable Contact Lookup Checkbox true Optionnel Active l'affichage des contacts dans les popups de marqueurs et la sélection de contacts lors de la planification de tournée.
Contact Query Fields Text Id,Name Optionnel Champs Contact à récupérer. Exemples : Id,Name,Email,Phone,Title
Contact Query Filter Text (vide) Optionnel Clause WHERE SOQL (sans le mot-clé WHERE). Ex : IsActive__c = true
Default Map Layer Picklist Standard Optionnel Fond de carte par défaut : Standard (rues), Terrain (topo), Satellite (vue aérienne).
Enable History Naming Checkbox true Optionnel Demande un nom de tournée avant l'enregistrement dans l'historique.
Last Visit Field Text (vide) Optionnel Nom API du champ Account stockant la date de dernière visite. Active le filtre "Non visité depuis". Ex : LastActivityDate ou Derniere_Visite__c
Not Visited Since (Days) Number (vide) Optionnel Nombre de jours par défaut pour le filtre "Non visité depuis".
Support Email Email (vide) Optionnel Email affiché sur l'écran de licence expirée. Si vide, contact@routeforce.app est utilisé.
💡 Configuration minimale : Pour une installation rapide, seules les clés dans Licence Config (étape 5) sont obligatoires. Tous les champs ci-dessus ont des valeurs par défaut sensées. Vous pouvez les ajuster après les premiers tests.

6.2 — Configuration par page (override App Builder)

Les propriétés configurables dans l'App Builder (étape 7) sont prioritaires sur les Custom Metadata. Cela permet d'avoir des configurations différentes par page (ex : Billing sur la page Account, Shipping sur une page dédiée).

7. Configuration App Builder

Étape 7.1 — Ajouter le composant à une page Lightning

  1. Ouvrez une page Lightning dans l'App Builder (page Account, Home ou App Page)
  2. Cherchez "routeforce" dans le panneau de composants (à gauche)
  3. Glissez-déposez le composant sur la page
  4. Configurez les propriétés dans le panneau de droite (voir ci-dessous)
  5. Cliquez sur Save puis Activate

7.2 — Propriétés configurables

💡 Filtres dynamiques : Les filtres s'adaptent automatiquement au type de champ (picklist → dropdown, booléen → toggle, multipicklist → sélection multiple). Vous pouvez filtrer sur n'importe quel champ standard ou custom de l'objet Account.

Étape 7.3 — Ajouter l'onglet RouteForce à la barre de navigation

Recommandé L'onglet RouteForce donne accès à la carte complète (vue tous comptes). Ajoutez-le à votre application Lightning :

  1. Allez dans Setup → App Manager
  2. Cliquez sur Edit sur votre application (ex: Sales)
  3. Onglet Navigation Items
  4. Ajoutez RouteForce dans les éléments sélectionnés
  5. Positionnez-le à l'emplacement souhaité et cliquez Save

Étape 7.4 — Ajouter l'onglet Map sur la fiche Account

Recommandé Pour voir la carte centrée sur un compte spécifique, ajoutez un onglet dans la page Account :

  1. Ouvrez la page Account dans l'App Builder
  2. Ajoutez un composant Tabs si absent, ou utilisez les onglets existants
  3. Créez un nouvel onglet nommé "Map" (ou "Carte")
  4. Glissez-déposez le composant RouteForce dans cet onglet
  5. Cliquez sur Save puis Activate

💡 Le composant se centre automatiquement sur le compte courant lorsqu'il est positionné sur une page Account. Les comptes à proximité sont visibles sur la carte, facilitant la planification de visites groupées.

9. Bonnes pratiques

Recommandation : champ Rollup "Dernière visite" Recommandé

Pour exploiter pleinement le filtre "Non visité depuis X jours" et identifier les comptes à revisiter, nous recommandons fortement de créer un champ Rollup Summary sur l'objet Account :

Option A — Rollup Summary (recommandé)

ParamètreValeur
LabelDernière Visite (ou Last Visit Date)
API NameLast_Visit_Date__c
TypeRollup Summary
Summarized ObjectEvents (activités liées au compte)
Roll-Up TypeMAX
Field to AggregateActivityDate (Date de l'événement)
Filter CriteriaSubject CONTAINS 'Visite' — ou filtrez par RecordType dédié si vous en avez un

Option B — Flow schedulé (alternative)

Si le Rollup Summary n'est pas possible (ex : filtres complexes, objets non-master-detail), créez un Scheduled Flow quotidien qui :

  1. Requête les Events des dernières 24h liés à des Accounts
  2. Met à jour Last_Visit_Date__c sur chaque Account concerné

Configuration dans RouteForce

Renseignez le nom API du champ dans RouteForce Config → LastVisitField__c (ex: Last_Visit_Date__c). Le filtre "Non visité depuis" sera alors disponible dans la carte.

Cas d'usage : identifier les comptes sans visite récente

Avec ce champ en place, vous pouvez :

  • 📍 Dans RouteForce : utiliser le filtre "Non visité depuis X jours" pour afficher uniquement les comptes à revisiter sur la carte
  • 📊 Dans un rapport/dashboard SF : créer un rapport sur les comptes où Last_Visit_Date__c < TODAY() - 30
  • 🔍 En SOQL (pour les admins) :
SELECT Name, BillingCity, Last_Visit_Date__c, OwnerId
FROM Account
WHERE Last_Visit_Date__c < LAST_N_DAYS:30
   OR Last_Visit_Date__c = NULL
ORDER BY Last_Visit_Date__c ASC NULLS FIRST

↑ Cette requête retourne tous les comptes non visités depuis 30 jours (ou jamais visités). Idéal pour alimenter une vue liste ou un dashboard de suivi commercial.

💡 Pourquoi un Rollup plutôt que LastActivityDate ? Le champ standard LastActivityDate inclut les appels, emails, tâches et tous types d'activités — pas uniquement les visites terrain. Un Rollup dédié (ou Flow) vous permet de filtrer précisément sur les visites physiques et d'identifier les comptes qui n'ont pas reçu de visite depuis N jours. C'est la base d'une bonne couverture terrain.

Recommandation : fonds de carte alternatifs

Pour activer les fonds Satellite (imagerie aérienne Esri) et Terrain (OpenTopoMap), ajoutez les CSP Trusted Sites correspondantes (voir étape 3.2). Sans ces sites autorisés, seul le fond Standard (OpenStreetMap) fonctionnera.

10. Dépannage

ProblèmeSolution
Écran "🔒 Licence expirée" Vérifiez que Licence Config → Default contient une clé de licence valide. Contactez contact@routeforce.app pour un trial.
Aucun marqueur sur la carte Vérifiez que vos comptes ont des coordonnées GPS (Setup → Data Integration Rules → Geocodes activés). Vérifiez le Address Field Prefix dans le CMDT.
Erreur "Optimiser la tournée" Vérifiez que l'ORS API Key est renseignée dans Licence Config et que le Remote Site Setting https://ors.routeforce.app est actif.
Composant non visible dans l'App Builder Vérifiez que le Permission Set RouteForce User ou RouteForce Admin est attribué à votre utilisateur.
Carte blanche (pas de tuiles) Ajoutez les CSP Trusted Sites pour https://*.tile.openstreetmap.org et https://unpkg.com (étape 3).
Fond Satellite ou Terrain ne charge pas Ajoutez les CSP Trusted Sites : https://server.arcgisonline.com (Satellite) et https://*.opentopomap.org (Terrain). Voir étape 3.2.
Erreur 401 sur les routes Votre clé API est invalide ou votre licence a expiré. Contactez le support.
Les filtres n'affichent rien Vérifiez que les Filter Fields configurés dans l'App Builder correspondent à des champs existants sur l'objet Account.
