1. Prérequis

ÉlémentRequis
Édition SalesforceEnterprise, Unlimited ou Developer
Version API≥ 62.0 (Winter '25)
Lightning ExperienceActivé (le package n'est pas compatible Classic)
Profil installateurSystem Administrator
Objet AccountChamps d'adresse géolocalisés (Billing et/ou Shipping avec Latitude/Longitude)
NavigateurChrome, Edge, Firefox ou Safari (version récente)
💡 Géolocalisation des adresses : Salesforce géolocalise automatiquement les champs d'adresse standard (BillingAddress, ShippingAddress) si l'option Data Integration Rules → Geocodes for Account Billing/Shipping Address est activée dans Setup → Data Integration Rules.

2. Installation du package

Étape 2.1 — Installer via le lien d'installation

Cliquez sur le lien correspondant à votre environnement :

EnvironnementLien d'installation
Production / Developer Installer en production
Sandbox Installer en sandbox

Étape 2.2 — Options d'installation

  1. Connectez-vous avec un profil System Administrator
  2. Sélectionnez "Install for Admins Only" (recommandé pour la configuration initiale)
  3. Cochez "Yes, grant access to these third-party web sites" si demandé
  4. Cliquez sur Install
  5. Attendez l'email de confirmation (quelques minutes)
⚠️ Package managé : Le namespace du package est routeforce. Tous les composants sont préfixés automatiquement (ex: routeforce__TourneeConfig__mdt).

3. Attribution des permissions

Étape 3.1 — Permission Set

Attribuez le Permission Set RouteForce User aux utilisateurs qui doivent accéder au composant :

Setup → Permission Sets → RouteForce User → Manage Assignments → Add Assignments

4. Configuration Custom Metadata

Le package utilise un type de métadonnées personnalisé routeforce__TourneeConfig__mdt pour la configuration. Chaque enregistrement contient un champ texte Value__c.

Setup → Custom Metadata Types → Tournee Config → Manage Records

4.1 — Enregistrements à créer

DeveloperName Label Value__c Description Obligatoire
ORS_VPS_Key ORS VPS Key (clé fournie par le support) Clé d'API pour le serveur VPS d'optimisation de tournées (ORS on-premise). Requise pour l'optimisation des itinéraires. ✅ Oui
ORS_API_Key ORS API Key (clé fournie par le support) Clé d'API HeiGIT (fallback cloud). Fournie avec la licence ou à obtenir sur openrouteservice.org. ⚡ Recommandé
Address_Field_Prefix Address Field Prefix Shipping ou Billing Champ d'adresse principal utilisé pour la géolocalisation. Fallback automatique : si l'adresse préférée n'a pas de coordonnées, l'autre type est utilisé. 📌 Optionnel (défaut : Shipping)
Contact_Lookup_Enabled Contact Lookup Enabled true ou false Active la sélection de contact lors de la création des RDV dans la tournée. 📌 Optionnel (défaut : false)
Contact_Query_Fields Contact Query Fields Id,Name,Email,Phone Champs Contact récupérés pour l'affichage dans la tournée. 📌 Optionnel
Contact_Query_Filter Contact Query Filter (clause WHERE SOQL) Filtre SOQL optionnel pour les contacts (ex: IsActive__c = true). 📌 Optionnel
💡 Smart fallback adresse : Si vous configurez Address_Field_Prefix = Billing, le système utilisera en priorité BillingLatitude/BillingLongitude. Si un compte n'a pas de coordonnées Billing, les coordonnées Shipping seront automatiquement utilisées comme fallback (et vice versa). Aucune donnée n'est perdue.

Étape 4.1 — Créer un enregistrement Custom Metadata

  1. Accédez à Setup → Custom Metadata Types
  2. Cliquez sur Manage Records à côté de "Tournee Config"
  3. Cliquez sur New
  4. Renseignez le Label et le DeveloperName (voir tableau ci-dessus)
  5. Renseignez le champ Value
  6. Cliquez sur Save

4.2 — Configuration de la clé ORS (obligatoire)

⚠️ Clé ORS VPS : La clé ORS_VPS_Key est nécessaire pour l'optimisation des tournées. Sans cette clé, le bouton "Optimiser" retournera une erreur. Contactez le support RouteForce pour obtenir votre clé.

5. Configuration App Builder

Étape 5.1 — Ajouter le composant à une page Lightning

  1. Ouvrez une page Lightning dans l'App Builder (page Account, Home, ou App Page)
  2. Cherchez le composant "RouteForce Locator" dans le panneau de gauche
  3. Glissez-déposez le composant sur la page
  4. Configurez les propriétés dans le panneau de droite

5.2 — Propriétés configurables

PropriétéDescriptionExemple
Address Field PrefixOverride du préfixe d'adresse (prioritaire sur le CMDT)Shipping ou Billing
Filter Field 1-12Champs Account à exposer comme filtresIndustry, Type
Type FieldChamp pour la colorisation des marqueursType
Type ValuesValeurs du champ type (séparées par virgule)Customer,Prospect,Partner
Marker FieldsChamps additionnels affichés dans le popupPhone,Website,Industry
Force Multi PicklistChamps à forcer en sélection multipleIndustry
💡 Hiérarchie de configuration : La propriété Address Field Prefix dans App Builder est prioritaire sur la valeur Custom Metadata. Cela permet de configurer un préfixe différent par page si nécessaire.

6. Test de validation post-installation

Checklist de validation

  1. ✅ Le package apparaît dans Setup → Installed Packages
  2. ✅ Le composant RouteForce Locator est disponible dans l'App Builder
  3. ✅ Les enregistrements Custom Metadata sont visibles dans Setup → Custom Metadata Types → Tournee Config
  4. ✅ Le composant se charge correctement sur une page Lightning (carte Leaflet visible)
  5. ✅ Les comptes avec adresses géolocalisées apparaissent comme marqueurs sur la carte
  6. ✅ Le bouton "Tournée" ouvre le panneau de planification
  7. ✅ L'optimisation fonctionne (requiert ORS_VPS_Key)
✅ Test rapide : Ouvrez la page Lightning contenant le composant, autorisez la géolocalisation dans le navigateur, et vérifiez que des marqueurs apparaissent autour de votre position. Cliquez sur un marqueur pour vérifier l'affichage des détails.

7. Dépannage

ProblèmeSolution
Aucun marqueur sur la carte Vérifiez que vos comptes ont des coordonnées GPS (Data Integration Rules activées). Vérifiez le Address_Field_Prefix.
Erreur "ORS_VPS_Key not configured" Créez l'enregistrement Custom Metadata ORS_VPS_Key avec la clé fournie.
Composant non visible dans l'App Builder Vérifiez que le Permission Set RouteForce User est attribué.
La carte ne se charge pas Ajoutez les CSP Trusted Sites suivantes dans Setup → CSP Trusted Sites :
https://ors.routeforce.app (callouts ORS)
https://*.openstreetmap.org (tuiles carte)
https://*.tile.openstreetmap.org
Erreur d'optimisation (timeout) Le serveur VPS est peut-être temporairement indisponible. Le système bascule automatiquement sur le fallback HeiGIT si ORS_API_Key est configurée.
