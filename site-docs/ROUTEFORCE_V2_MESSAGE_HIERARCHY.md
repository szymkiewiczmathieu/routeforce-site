# RouteForce v2 — Message Hierarchy

Updated: 2026-04-02
Source basis: current root site + validated product claims confirmed by routeforce-dev.

## 1. Core promise

**Plan routes, create visits, execute in the field, and complete reporting — without leaving Salesforce.**

Expanded version:
RouteForce helps field teams plan routes, create visits and events, execute check-in/check-out, and complete visit reporting inside Salesforce, with fixed org pricing instead of per-user licences.

## 2. The three repeatable proof pillars

### Pillar 1 — Salesforce-native end-to-end flow
- Native Salesforce app
- No external field tool
- Route planning, visit creation, check-in/check-out, and reporting in one system

### Pillar 2 — Fixed org pricing
- €599/month excluding tax for the whole org
- No per-user pricing
- Easier rollout and easier internal justification

### Pillar 3 — Field execution inside Salesforce
- Visit planning and route planning
- Event creation from planned routes / visits
- Configurable visit reports
- Mobile use and nearby-account workflows

## 3. Message guardrails

### Confirmed and safe to push
- Native Salesforce app / no external tool
- Fixed org pricing / no per-user pricing
- Route planning in Salesforce
- Visit planning in Salesforce
- Field execution workflow in Salesforce
- GPS-verified check-in (500m radius)
- Visit reports with configurable fields
- Optional Quick Text support in visit report notes
- Nearby accounts / GPS proximity workflow
- Event creation from planned routes / visits
- Mobile use inside Salesforce mobile

### Must be nuanced
- Previous visit context shown automatically
- Better territory coverage / more useful visits / less drive time (frame as likely outcomes, not guaranteed performance claims)

### Avoid on the site
- Any wording implying fully automatic prior-context surfacing everywhere in the UX
- Any guaranteed ROI / mileage / productivity claim without proof
- Any wording that suggests Quick Text is universal across the whole reporting flow rather than optional support in notes
