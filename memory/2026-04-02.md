## Freemium funnel strategy (Boss)
- RouteForce site CTA/source-of-truth funnel: encourage prospects to install the app first via the freemium entry point, then move them to the trial, then convert them to the paid full version.
- Do not blindly normalize all CTAs toward `#contact`. Audit and optimize pages against the real funnel: Install freemium -> Trial -> Full version.
- When reviewing blog/docs/site conversion paths, AppExchange install CTAs can be correct if they support the freemium-first motion.

## Lesson learned
- When updating published documentation version labels, do not stop at `docs/index.html` or link text. Also update visible version mentions inside `docs/setup-guide.html`, `docs/user-guide.html`, and `docs/configuration-guide.html` before claiming done.
- Verify with grep that the old version string no longer appears in the published guide HTML files.

- Durable lessons from 2026-04-02 site work:
  - Do not announce documentation/version updates as done until verifying both the docs hub and all three published guides (`docs/index.html`, `setup-guide.html`, `user-guide.html`, `configuration-guide.html`) actually show the new public version.
  - Current public funnel/source of truth for RouteForce site messaging: `Install RouteForce from AppExchange -> Start with the free app -> Unlock premium features with a RouteForce licence`. Avoid inventing or defaulting to `#contact` or `trial -> full paid` unless explicitly confirmed.
  - Public version truth evolved during the day: first uncertainty, then explicit confirmation from user that public baseline became `7.6.0`, later `RouteForce V7.7.0-1 — RELEASED`; site/public essentials were aligned to V7.7.0.
  - Important behavioral correction: when auditing, separate technical correctness from product strategy correctness. Earlier mistake was pushing a generic contact/trial funnel that conflicted with the actual freemium strategy.
  - Visual QA lesson: homepage fixes are not enough; mobile header/CTA regressions must be checked across all public templates (home, blog, docs, pricing, secondary pages), not just `index.html`.
  - End-of-day known remaining issue after browser recheck: `pricing.html` mobile/public copy still uses old `14-day free trial` messaging and needs alignment with the validated freemium/public baseline.
