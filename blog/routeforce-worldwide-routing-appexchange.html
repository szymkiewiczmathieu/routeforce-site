RouteForce V7.4: worldwide routing engine and AppExchange readiness

Version 7.4 brings a global routing engine covering 309 million road nodes, the start of our AppExchange security review, and a batch of features that make RouteForce a more complete field operations platform inside Salesforce.

Worldwide routing: 309M nodes, 387M edges

Until now, RouteForce routing was scoped to a single country per instance. V7.4 removes that limitation entirely. The routing engine now covers 309 million nodes and 387 million edges from the OpenStreetMap road network, processed through OpenRouteService.

This means a sales manager in France can plan routes for reps traveling across Belgium, Germany, or Spain in a single session, without switching configurations or dealing with border gaps in the road graph.

Self-hosted, GDPR-compliant infrastructure

The worldwide routing instance runs on OVHcloud Gravelines (Hauts-de-France). All route computations happen on European soil, on servers operated by a European cloud provider. No waypoint data, no intermediate coordinates, and no route geometry leave the EU. This is a straightforward GDPR compliance story for customers who need it.

The server holds the full planet graph in memory and responds to route requests in real time. There is no dependency on Google, Mapbox, or any US-based routing API for the actual route calculation.

AppExchange security review: in progress

RouteForce has entered the Salesforce AppExchange security review process. This is the formal evaluation that every managed package must pass before it can be listed on the AppExchange marketplace.

What the review covers

  • Static code analysis — Salesforce scans all Apex classes, triggers, Visualforce pages, and LWC components for security vulnerabilities including SOQL injection, XSS, CSRF, and insecure data handling.
  • Architecture review — how the package handles data isolation between orgs, how it communicates with external services (Named Credentials, callout patterns), and whether it respects platform limits.
  • Data handling — what data leaves the org, where it goes, how it is stored, and whether users have visibility into that flow.
  • Permission model — whether the package requests only the permissions it needs and whether it enforces object- and field-level security correctly.

What this means for customers

Once the review is complete, RouteForce will carry the Salesforce security reviewed badge. For IT and procurement teams, this reduces the due diligence burden: Salesforce has independently verified the package meets their security baseline. For now, the review is actively in progress and we will update this page when it completes.

You can already find the listing on AppExchange: RouteForce on AppExchange.

New features in V7.4

Heatmap overlay

Managers can now toggle a heatmap layer on the map to visualize record density, visit frequency, or any numeric field. This makes it easy to spot under-served territories or clusters of high-value accounts at a glance, without exporting data to a BI tool.

Bulk select with the lasso tool

Drawing a freeform shape on the map now selects all records inside it. From there, reps can add selected records to a route, assign them to a colleague, or run a bulk action. This replaces the tedious one-by-one selection workflow that was the only option in earlier versions.

Saved and shared filters

Filter configurations can now be saved and shared across users. A manager can define a filter like "accounts with no visit in the last 30 days and revenue above 50K" and share it with the entire team. Filters are stored as Salesforce records, so they respect the org's sharing model.

Visit report with custom fields

The visit report form now supports custom fields. Out of the box, V7.4 includes:

  • Customer satisfaction — a picklist or rating scale captured on the spot
  • Order amount — a currency field for orders placed during the visit
  • Next visit date — a date field to schedule the follow-up immediately

Admins can add more fields to the visit report layout without code, using standard Salesforce page layout configuration.

Quick text for visit notes

Field reps can now use Salesforce Quick Text to insert predefined text snippets into visit notes. This speeds up note-taking for repetitive observations ("shelf display compliant", "out of stock on SKU X") and standardizes the data that flows back into reports.

Usage tracking dashboard

A new dashboard gives admins visibility into how the team uses RouteForce: number of routes created, visits logged, map sessions, and average route length. This helps justify the investment and identify reps who may need additional training.

3 permission sets: Admin, Manager, User

V7.4 ships with three pre-built permission sets that map to common field sales roles:

  • RouteForce Admin — full configuration access, usage dashboard, custom field management
  • RouteForce Manager — team route visibility, shared filters, heatmap overlay, reassignment
  • RouteForce User — personal route planning, visit reports, quick text, map access

This eliminates the need to build custom permission sets from scratch during installation.

What's next

The V7.4 release sets the foundation for several initiatives planned for Q2 and Q3 2026:

  • AppExchange listing completion — once the security review passes, RouteForce will be installable directly from the AppExchange with one click.
  • Territory optimization — automated territory balancing based on workload, travel time, and account value.
  • Offline route access — cached route data for reps working in areas with poor connectivity.
  • Deeper Flow integration — more pre-built Flow templates for common field scenarios (prospection blitz, delivery confirmation, audit checklist).
